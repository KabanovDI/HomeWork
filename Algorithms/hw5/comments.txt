1) Для проверки, является ли бинарное дерево полным проверяем, что:
- Все уровни, кроме возможно последнего, заполнены полностью.
- На последнем уровне все узлы сдвинуты влево.
Для такой проверки идеально подходит обход в ширину, идём по уровням слева направо и если встречаем nullptr, то после него не должно быть ни одного не-nullptr узла, и так пока не закончится очередь.

2) Инициализация кучи: O(K log K) — вставка K элементов.
Для каждого из N элементов:
- Извлечение минимума: O(log K)
- Вставка следующего элемента: O(log K)
Итого: O(K log K + 2 N log K) = O(N log K) (так как N обычно значительно больше K)
Пространственная сложность: O(K) + O(N)
O(K) — размер мин-кучи (максимум K элементов одновременно)
O(N) — результирующий массив 

3) Время: O(N log K), каждый элемент необходимо добавить в кучу.
Память: O(K) для кучи, храним только такой длины.

4) In-order обход естественно дает отсортированный порядок. Прерываем обход как только нашли K-й элемент.

5) Определение высоты: Высота узла - это длина самого длинного пути от этого узла до листа.
Высота листа = 0 (или -1 если считать высоту пустого дерева за -1)
Высота пустого узла = -1
Метод вычисления: Используем post-order обход (сначала дети, потом родитель)
Рекурсивно вычисляем высоты левого и правого поддеревьев
Высота текущего узла = max(левая_высота, правая_высота) + 1
Такой подход гарантирует, что высоты детей известны до вычисления высоты родителя
Баланс-фактор: Разница между высотами правого и левого поддеревьев
Положительное значение: перевес вправо
Отрицательное значение: перевес влево
Ноль: идеально сбалансировано

6) Почему алгоритм корректен?
Воспользуемся методом индукции для доказательства:
Базовый случай: Пустое дерево (nullptr) остается пустым - зеркальное отражение пустого дерева это пустое дерево.
Индукционный шаг: Предположим, что алгоритм корректно отражает левое и правое поддеревья. Тогда для получения зеркального отражения всего дерева достаточно поменять местами уже отраженные левое и правое поддеревья.
Временная сложность: O(N) - каждый узел обрабатывается ровно один раз.

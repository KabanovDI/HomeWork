1) Храним переменную с максимальной длинной последовательности и длиной последовательности в данный момент, проходимся по массиву и инкрементируем вторую переменную если обрабатываемый элемент больше предыдущего, иначе возвращаем её значение к единице, перед этим сравнив с максимальной длиной последовательности, в случае если та меньше - обновляем значение.

2) Элемент[i][j] = элемент[i-1][j-1] + элемент[i-1][j]. Элементы на склонах треугольника - базовые случаи, равны единице. Сложность алгоритма: 1 + 2 + ... + n = n+1 * n/2 = O(n^2).

3) Благодаря выделению дополнительной памяти для хранения промежуточных значений рекурсивная функция работает за O(n). Рекурсия основана на мат индукции, поэтому корректна. База: сумму ноль можно собрать из нуля элементов, отрицательную сумму никак нельзя. Шаг: минимум из вариантов для сумм n - ci (ci - номиналы монет) будет ответом с минус единицей для суммы n, а если все эти предыдущие варианты невозможны, то и для суммы n невозможно подобрать комбинацию монет.

4) Используем подход с мемоизацией промежуточных значений, наше dp - двумерный массив, значения dp[i][j] в котором равны true или false в зависимости от того, является ли подстрока от i-ого до j-ого элемента палиндромом. Заполняем мы её постепенно по диагоналям, начиная с главной и кончая правым верхним элементом. При заполнении мы сравниваем всего два символа строки и проверяем нужное значение dp из ранее заполненных, поэтому сложность O(n^2). При этом главная диагональ вся true - базовый случай. Затем проходимся по элементам dp в обратном порядке (самые длинные подстроки расположены ближе к верхнему правому углу) и используем первое попавшееся true для вывода нужной подстроки. Поиск тоже O(n^2). Итого: временная и пространственная сложности - O(n^2).
